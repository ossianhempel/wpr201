Processanalys

Övergripande

I det här projektet har jag valt att använda CSS-variabler för att lagra de vanligaste färgerna. Detta är ett effektivt sätt att hantera färger eftersom det gör det enkelt att uppdatera dem genom hela dokumentet om det behövs (Wilkins, 2022). Det underlättar även att göra färganvändningen på hemsidan konsekvent eftersom färgerna enkelt återanvänds.

Box-modellen appliceras på alla element i dokumentet för att säkerställa att bredd och höjd inkluderar padding och border. Detta leder till bättre kontroll över elementens storlek och placering på sidan. Att applicera box-modellen på alla element ger en enhetlig och förutsägbar layout där padding och border ingår i beräkningen av elementens totala storlek (Coyier, 2021a). Alternativt kan man använda den traditionella box-modellen, men detta kan leda till svårigheter när man anpassar layouten för olika skärmstorlekar och ger mindre kontroll över elementens storlek och placering.

Jag utgick från de mock-ups som gavs i uppgiften och använde en desktop-first-strategi för att den hade mest detaljer och tanken var då att det skulle bli lättare att skala ner och anpassa sidan för mindre skärmar senare efter att ha skapat en mer komplex layout för desktop. Ett alternativ är en mobile-first-strategi, men detta kan göra det svårare att skapa en rik och detaljerad desktop-layout från en enklare mobilversion. Framförallt då jag visste om hur designen skulle se ut baserat på mockupsen (Johnson, 2019).

Sidan delas in i tre primära sektioner med hjälp av CSS Grid. CSS Grid används för att skapa sidans grundläggande layout, vilket ger en hög grad av flexibilitet och kontroll över elementens placering och storlek (House, 2023). Ett alternativ till CSS Grid är Flexbox, men CSS Grid är bättre på att skapa layouts för hela sidor, både praktiskt och prestandamässigt (Rendle, 2017). Jag använde istället flexbox för att positionera elementen inom den större layouten just för att det är smidigt och flexibelt, men CSS Grid kan vara ett bättre alternativ för att skapa den övergripande layouten då det ger mer kontroll.

Wrapper

Jag valde att använda en wrapper för att centrera och innesluta innehållet på sidorna. Wrappern skapades genom ett `div` element som jag gav id “wrapper” (Kosev, 2017). Man kan lika gärna använda en class istället men eftersom jag bara använder en wrapper i dokumentet passade det med id. Ett alternativt sätt är att använda ett container element men det är mest en smaksak och Kosev (2017) rekommenderar att använda den metod jag använde. Jag applicerade en max-width på wrapper för att undvika att innehållet sträcker sig utöver kanterna och man undviker på så vis en horisontell scrollbar när webbläsarfönstret är väldigt smalt (Kosev, 2017).

Header

Headern fick en bakgrundsfärg och bakgrundsbild som täcker hela headern, enligt instruktionerna. Bakgrundsbilden stylas genom CSS och mer specifikt “background-size: cover” vilket gör att bilden skalar och anpassar sig efter headerns dimensioner (Coyier, 2022). I headern finns även logotyp, en navigationsmeny och en kortare text. Flexbox används för att placera innehållet vertikalt och centrera det. Flexbox gör headern responsiv så att den anpassar sig för olika skärmstorlekar och orientationer, exempelvis på mobila enheter (Niederst Robbins, 2018). Ett alternativ är att använda CSS Grid eller absolut positionering, men Flexbox erbjuder en mer flexibel och lättanvänd lösning för centrerade och responsiva layouter i den här situationen (Coyier, 2021b). Självbeskrivningen centrerar innehållet med en maxbredd och en bakgrundsfärg med viss transparens för att göra texten lättare att läsa. Jag använder semantiska HTML element såsom `header`, `nav` och `ul` för att göra hemsidan mer tillgänglig.

Main

Jag skapade “huvudsektionen” genom en kombination av Flexbox och individuella innehållsboxar för att uppnå den önskade layouten. Detta resulterade i en flexibel och anpassningsbar layout som fungerar i flera skärmstorlekar.I HTML skapade jag ett `main` element som innehåller en `div` med klassen “contentboxes” som innehåller tre individuella `div`-element med klasserna “contentbox”. Varje contentbox innehåller en bild och en text där texten är omsluten genom ytterligare en `div` med klass “image-circle” för att skapa den cirkulära ramen åt bilderna. Den här strukturen hjälper mig separera innehållet och hålla koden tydlig. I CSS använde jag Flexbox för `main` och innehållboxarnas kontainer för att enkelt kunna positionera och hålla isär innehållsboxarna utan att behöva använda manuella uträkningar och justeringar. Dessutom blir det då responsivt för olika skärmstorlekar.

Jag skapade de cirkulära bilderna genom en kombination av ett cirkulärt element (class=”image-circle”) och de faktiska bildelementen inom de cirkulära elementen. Jag gav dem en fast höjd och bredd, samt en border-radius för att göra de cirkulära. `Overflow: hidden” användes sedan för att dölja den del av bilderna som var större en det cirkulära elementen. På så sätt visades bara den del av bilden som fick plats i cirkeln. För att positionera bilderna enligt mockupen (centrera den mittersta bilden och höger-/vänsterjustera bilderna på sidorna på deras innehållsboxar) använde jag float. Det hade även gått att använda Flexbox här men float blev lättast då jag bara behövde applicera önskad float på varje bild utan att behöva justera något runtomkring. För att skapa överlappning mellan bilderna och ovan element, som på mockupen, använde jag negativ margin. Eftersom margin vanligtvis skapar utrymme mellan två element gör negativ margin att de överlappar istället (Imbong, 2009).

Footer

Footern delades upp i tre sektioner som innehåller kontaktinformation och adress. Varje sektion är en `div` med ett `h3` element för rubriken och `p` som innehåller informationen. Man hade även kunnat arbeta med listor här men eftersom två av tre element bara innehåller en paragraf under rubriken gick det snabbare att använda `p` och slutresultatet blev detsamma. I CSS:en används återigen Flexbox just för att det underlättar positioneringen av elementen och samtidigt gör dem responsiva utan ytterligare åtgärd. Likt headern, har footern en bakgrundsbild som ställdes in med “bottom” för att matcha headern. De använder en likadan bild men genom att positionera så footern endast visar botten av bilden medan headern visar en del högre upp på bilden skapas en sammanhållning då bilderna “matchar”. Inom de individuella `div`-elementen i footern använder jag `flex-direction: column` så att titeln och detaljerna positioneras vertikalt. Paragraf-elementen inom footern har en margin satt till 0 för att ta bort automatiskt spacing och ger på så sätt mer kontroll över layouten.

Media Queries

Surfplatta

Den första media queryn är till för surfplattor och gäller för bredd mellan 768px och 1024px. 768px är en standardbredd för just surfplattor (Mutua, 2022). För denna storlek sätter jag `.contentboxes` till `flex-wrap: wrap` så att boxarna kan flyttas ned till “nästa rad” om det krävs för att få plats (Fitzgerald, 2021). Jag satte även en responsiv bredd på boxarna till 50% av det tillgängliga utrymmet minus 1.5 viewport-bredd. Detta resulterar i en lagom storlek som anpassar sig efter sidans bredd. Jag ger även `margin-bottom` en gräns på minimalt 100px vilket hjälper till att hålla ett rimligt mellanrum när boxarna flyttas ner. `float`-värdena sätts till `none` för att låta boxarna flytta på sig utifrån tillgängligt utrymme.

Mobil

Eftersom jag har en storlek för tablet lät jag alla storlekar under 768px ha den mobila designen, vilket alltså får en maxbredd på 767px (Mutua, 2022). Här har jag ändrat `.header-content`’s `flex-direction: column` så att boxarna positioneras vertikalt ovanför varandra istället för horisontellt. Detta då flera boxar inte får plats på den mobila storleken. Logotypens storlek breddas och centreras för att matcha den mobila layouten bättre. För de cirkulära bilderna tas float bort och ordningen på boxarna sätts manuellt genom `order` för att positionera den mittersta boxen längst upp på den mobila sidan enligt instruktionerna (Monus, 2021). Utan den här justeringen skulle den vänstra boxen visas längst upp på den mobila versionen.

Print

Den tredje och sista media queryn justerar vad som visas när man skriver ut sidan. Jag döljer bakgrundsbilden för `header` och `footer` då de större bilderna skulle döljas enligt uppgiftsinstruktionerna. Jag döljer även menyn eftersom man ändå inte kan interagera med den utskriva sidan så det är onödigt att inkludera interaktiv element. De mindre bilderna som stannar kvar vid utskrift är satta till gråskala för att ta bort färger och på så sätt spara bläck och kompabilitet. I `footer` använder jag pseudo-elementet `::after` för att addera en copyright-text efter de andra elementen i footern.
